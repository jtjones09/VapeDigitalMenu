CREATE INDEX IF NOT EXISTS idx_products_name_trgm

SELECT 
  product_name,
  SIMILARITY('SMOK Nord 4', product_name) as score
FROM products

ORDER BY score DESC
LIMIT 5;

```
## 🔎 PART 2: CREATE DUPLICATE SEARCH API ENDPOINT

**File:** `/server/routes/products.ts`

**NEW ENDPOINT:**
```
POST /api/products/search-duplicates
Body: {
  brandId: string (optional - if using existing brand),
  brandName: string (optional - if using custom brand name),
  productName: string (required),
  productType: string (optional - helps narrow results)
}

Response: {
  matches: [
    {
      id: uuid,
      productName: string,
      productType: string,
      brandName: string,
      similarity: number (0.0 to 1.0),
      isCustom: boolean,
      variantCount: number
    }
  ]
}
Implementation logic:

Search ONLY global products (is_custom = false)
Use fuzzy matching on product_name (SIMILARITY function)
Match by brand (either exact brand_id match OR fuzzy brand name match)
Return top 5 results ordered by similarity score
Include similarity threshold of 0.6 (60% match minimum)
Join with brands table to get brand name
Include variant count for context

Key SQL concepts to use:
sql-- Similarity function (returns 0.0 to 1.0)
SIMILARITY(column_name, 'search term')

-- Example query structure
SELECT 
  p.*,
  COALESCE(b.brand_name, p.custom_brand_name) as brand_name,
  SIMILARITY(p.product_name, $searchTerm) as similarity,
  COUNT(pv.id) as variant_count
FROM products p
LEFT JOIN brands b ON p.brand_id = b.id
LEFT JOIN product_variants pv ON pv.product_id = p.id
WHERE p.is_custom = false
  AND SIMILARITY(p.product_name, $searchTerm) > 0.6
  AND (p.brand_id = $brandId OR SIMILARITY(brand_name, $brandName) > 0.6)
GROUP BY p.id, b.brand_name
ORDER BY similarity DESC
LIMIT 5
Performance considerations:

Use the GIN indexes created in Part 1
Limit results to 5 maximum
Only query when productName is at least 3 characters

⏱️ PART 4: IMPLEMENT REAL-TIME DUPLICATE SEARCH
In the custom product form component:
State to add:
typescriptconst [matches, setMatches] = useState([]);
const [isSearching, setIsSearching] = useState(false);
Debounced search logic:
When user types in productName field (or changes brand), trigger search after 500ms delay:
typescript// Debounce search - only search after user stops typing for 500ms
useEffect(() => {
  // Only search if we have minimum required info
  if (productName.length < 3) {
    setMatches([]);
    return;
  }
  
  setIsSearching(true);
  
  const timer = setTimeout(async () => {
    try {
      const response = await fetch('/api/products/search-duplicates', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          brandId: brandId,
          brandName: customBrandName,
          productName: productName,
          productType: productType,
        })
      });
      
      const data = await response.json();
      setMatches(data.matches || []);
    } catch (error) {
      console.error('Duplicate search failed:', error);
    } finally {
      setIsSearching(false);
    }
  }, 500);
  
  return () => clearTimeout(timer);
}, [brandId, customBrandName, productName, productType]);
Why 500ms debounce:

User types "S-M-O-K" → Only 1 search after they finish
Not 4 separate searches (better performance)
Feels responsive but doesn't spam the API

Why minimum 3 characters:

Prevents searching on "S" or "SM" (too vague)
Reduces unnecessary API calls
