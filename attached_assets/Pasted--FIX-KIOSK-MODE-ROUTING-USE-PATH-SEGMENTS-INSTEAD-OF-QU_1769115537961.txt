# FIX KIOSK MODE ROUTING - USE PATH SEGMENTS INSTEAD OF QUERY PARAMS

## ⚠️ CRITICAL SAFETY RULES

DO NOT:
- Delete any existing components
- Break the guided navigation flow
- Remove any features
- Truncate any code

DO:
- Change kiosk mode from query param (?mode=kiosk) to path segment (/kiosk/)
- Update ALL navigation to preserve kiosk mode
- Update ALL route definitions
- Test thoroughly

---

## THE PROBLEM

Current URLs use query params:
- /menu/abc123?mode=kiosk
- /menu/abc123/regular?mode=kiosk

Wouter captures query strings incorrectly, breaking navigation.

## THE SOLUTION

Use path segments instead:
- /menu/kiosk/abc123
- /menu/kiosk/abc123/regular
- /menu/kiosk/abc123/regular/fruit

This makes kiosk mode part of the route path, not a query param.
