FIX #7: CREATE DATABASE VIEW FOR EASY QUERYING
Create a view that handles brand name logic automatically:
sqlCREATE OR REPLACE VIEW products_full AS
SELECT 
  p.*,
  -- Brand name (either from brands table or custom field)
  COALESCE(b.brand_name, p.custom_brand_name, 'Unknown Brand') as brand_name,
  b.logo_url as brand_logo_url,
  b.website as brand_website,
  -- Shop info (for custom products)
  s.shop_name as created_by_shop_name
FROM products p
LEFT JOIN brands b ON p.brand_id = b.id
LEFT JOIN shops s ON p.created_by_shop_id = s.id;
Usage in queries:
typescript// Get all products available to shop123
const products = await db.execute(sql`
  SELECT * FROM products_full
  WHERE is_custom = false  -- Global products
     OR created_by_shop_id = ${shopId}  -- Their custom products
  ORDER BY brand_name, product_name
`);

// brand_name is already computed (no complex COALESCE needed)
This view works for BOTH custom and global products seamlessly.

FIX #8: UPDATE API LOGIC FOR "AVAILABLE PRODUCTS"
File: /server/routes/products.ts
When shop browses products to add to their menu, show:
typescriptrouter.get("/api/products/available", async (req, res) => {
  const shopId = req.user.shopId;
  
  const products = await db
    .select()
    .from(products)
    .where(
      or(
        eq(products.isCustom, false),  // All global products
        eq(products.createdByShopId, shopId)  // Their custom products
      )
    )
    .orderBy(products.productName);
  
  res.json(products);
});
Key point: Shop sees:

ALL global products (available to everyone)
ONLY their own custom products (not other shops' custom products)
